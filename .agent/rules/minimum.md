Правило переиспользования и минимизации компонентов

Не нужно создавать новые компоненты без необходимости.

Если логика и назначение похожи
— необходимо адаптировать существующий компонент,
а не создавать новый.

Если логика отличается существенно
— допускается создание нового компонента,
но с обязательной минимизацией количества компонентов.

Перед созданием нового компонента обязательно:

проверить существующие компоненты

оценить возможность:

расширения через props

изменения поведения через config

композиции из уже имеющихся компонентов

Запрещено:

дублировать компоненты с незначительными отличиями

создавать копии «на всякий случай»

плодить компоненты ради удобства одной страницы

Один компонент — одна ответственность.
Одна логика — один компонент.
